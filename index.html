About CSS

CSS (Cascading Style Sheets) is a fundamental web technology used for styling and formatting web pages. It separates the content from the presentation, allowing developers to control the layout, colors, fonts, and other visual aspects of a website. CSS simplifies website maintenance and promotes consistency across multiple web pages.

CSS plays a crucial role in modern web development by enabling developers to style and design web pages, ensuring visual consistency, responsiveness, and enhanced user experience.

Basic code examples

Here are three basic CSS code examples with explanations:

This code targets an HTML element with the class name "my-text" and sets its text color to blue. The color property controls the color of the text within the selected element 
.my-text {
        color: blue;
      }
This code sets the background color of the entire webpage's body to a light gray shade (#f2f2f2). The background-color property defines the background color of an element. 
body {
       background-color: #f2f2f2;
      }
"This code targets all <h1> heading elements and specifies their font size as 24 pixels and the font family as Arial or any sans-serif font available on the user's device. The font-size property controls the size of the font, and the font-family property defines the font face to be used." 
h1 {
     font-size: 24px;
     font-family: Arial, sans-serif;
     }
CSS and SASS

CSS (Cascading Style Sheets) is a standard web technology used for styling web pages. SASS (Syntactically Awesome Style Sheets) is a CSS preprocessor that adds additional features to CSS, such as variables, nesting, mixins, and functions, to make styling more efficient and maintainable. SASS code is compiled into CSS before deployment.

Main differences of CSS and SASS

  • Syntax: CSS uses a straightforward syntax with selectors, properties, and values, while SASS introduces additional features like variables, nesting, mixins, and functions, making the code more organized and reusable.
  • Compilation: CSS is interpreted by the browser directly, while SASS requires a compilation step. SASS files need to be compiled into CSS before they can be used on the web.
  • Reusability: SASS allows for the creation of reusable code snippets called mixins, which can be included in multiple places, reducing redundancy and improving code maintainability. CSS lacks this feature, requiring duplication of code in such scenarios.

CSS Selectors

CSS selectors are used to target specific HTML elements and apply styles to them. They allow developers to select elements based on their tag name, class, ID, attributes, and hierarchical relationships.

Tag Selector: 
p {
  color: red;
  }
Class Selector: 
.highlight {
  background-color: yellow;
  }
Id Selector: 
#green-text {
  color: green;
  }
Box Model
The box model is a fundamental concept in CSS that defines how elements are structured and rendered on a webpage. It consists of four layers: content, padding, border, and margin.

Content: The content area represents the actual content of an element, such as text, images, or other HTML elements. Its dimensions are defined by properties like width and height.

Padding: The padding area surrounds the content and provides space between the content and the element's border. It can be adjusted using properties like padding-top, padding-right, padding-bottom, and padding-left.

Border: The border is a line that surrounds the padding and content areas. It can be styled using properties like border-width, border-style, and border-color.

Margin: The margin area creates space around the element and separates it from other elements. It can be controlled using properties like margin-top, margin-right, margin-bottom, and margin-left.

Media Queries
The box model is a fundamental concept in CSS that defines how elements are structured and rendered on a webpage. It consists of four layers: content, padding, border, and margin.

Media queries are a CSS feature that allows developers to apply different styles based on the characteristics of the device or viewport size. This enables responsive design, where webpages can adapt their layout and appearance to different screen sizes or devices.

Media queries use specific conditions, such as screen width, device orientation, pixel density, or media types, to determine when a particular set of styles should be applied.

Here's an example of a media query that adjusts the font size of a paragraph when the screen width is less than or equal to 600 pixels:

@media screen and (max-width: 600px) {
  p {
   font-size: 14px;
     }
  }
